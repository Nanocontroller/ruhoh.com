So How is ruhoh Better?

Universal Static Blog API

ruhoh's main vision is to make it easier for all developers to share their knowledge.

Ruhoh’s Universal Blog API specification allows every developer, regardless of preferred language and platform to join the community.

The Universal Blog API provides standardized endpoints, allowing an implementation to interact with the blog regardless of language.

ruhoh beta is written in ruby but PHP, node.js, and Python are on the Roadmap to start.

Language Agnostic

ruhoh modules can be written in any language by using the The Universal Blog API endpoints.

Templating will be the hardest to maintain consistency over but thanks to Mustache we get a solid multi-language head-start.

Modular

Ruhoh uses a modular architecture as outlined in how ruhoh works. Modularization decouples the main ruhoh components, allowing you to customize or replace individual components as needed.

Templaters, converters, parsers, compilers, even the command-line tool can be extended or simply swapped out - ruhoh is made to be customized.

Made for Designers (kinda)

ruhoh provides modular, cascading, theming support natively. Nearly every structural and design component can be name-spaced relative to a theme.

Cascading files allow theme specific implementations to override default and system level code.

A client-side previewer is on the roadmap. Designers can develop and preview themes with zero server-side language dependencies.
